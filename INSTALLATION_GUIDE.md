## 🎯 OPTION A: Install WITHOUT systemd (RECOMMENDED)

**Best for:** Getting started quickly, testing, development

### Installation Steps:

```bash
# 1. Navigate to project
cd /mnt/d/Dev/whisperflow-cloud

# 2. Run installer (NO systemd)
./install_no_systemd.sh

# 3. Start WhisperFlow

```

### How Auto-Start Works:
- Adds startup script to `~/.bashrc`
- WhisperFlow starts automatically when you open a new terminal
- No systemd required

### Usage:
```bash
whisperflow start     # Start everything
whisperflow stop      # Stop everything
whisperflow status    # Check status
whisperflow logs      # View logs
whisperflow restart   # Restart everything
```

## 🔧 OPTION B: Enable systemd (Advanced)

**Best for:** Production use, systemd-based workflows

### Step 1: Enable systemd in WSL2

Edit WSL configuration:
```bash
sudo nano /etc/wsl.conf
```

Add this content:
```ini
[boot]
systemd=true
```

Save and exit (Ctrl+X, then Y, then Enter)

### Step 2: Restart WSL2

**From PowerShell (Windows):**
```powershell
wsl --shutdown
```

**Then reopen your WSL2 terminal**

### Step 3: Verify systemd is running

```bash
ps -p 1 -o comm=
# Should output: systemd
```

### Step 4: Run original installer

```bash
cd /mnt/d/Dev/whisperflow-cloud
./install.sh
```

## 🐛 Troubleshooting

### Option A (NO systemd)

**Client not starting:**
```bash
# Check client log
cat /tmp/whisperflow-client.log

# Restart manually
whisperflow restart
```

**Auto-start not working:**
```bash
# Check if startup script exists
ls -la ~/.whisperflow_startup.sh

# Check if added to .bashrc
grep "whisperflow_startup" ~/.bashrc
```

### Option B (WITH systemd)

**systemd still not working after restart:**
```bash
# Verify systemd is PID 1
ps -p 1 -o comm=
# Should say: systemd

# If not, check /etc/wsl.conf
cat /etc/wsl.conf
# Should have [boot] systemd=true
```
